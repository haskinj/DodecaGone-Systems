CR-IMRaD (Cognitive-Resonant IMRaD) is a writing standard that converts dense technical knowledge into documents anyone can read — without sacrificing scientific fidelity.

> **Suggested GitHub Topics:** `writing-standard`, `scientific-communication`, `technical-writing`, `cr-imrad`, `open-source-standard`

---

## Simple Summary
Research papers are hard to read. Not because the ideas are bad, but because the writing creates unnecessary friction. 

**CR-IMRaD fixes this by forcing authors to:**
1. Open with a physical metaphor before introducing technical terms.
2. Compress their thesis into a bolded skimmable path.
3. Place citations on the exact same page as their references. 

The standard defines sentence length limits, visual integration rules, and a three-tier automation framework (GREEN/AMBER/RED) for building conversion tools. It has been tested on papers from quantum cognition, clinical wellness, and theoretical neuroscience and confirmed as domain-independent.

## Automation Tiers
- 🟢 **GREEN:** Fully automatable (margins, fonts, pagination, citation numbering, sentence length checks).
- 🟡 **AMBER:** Machine suggests, author confirms (speculative bolding, opening metaphor, Simplest Summary draft).
- 🔴 **RED:** Author must decide (final bold confirmation, humor judgment, conflict of interest).

---

## Key Principles
1. You may NOT introduce a technical concept using technical terms.
2. The skimmable path (bolded words) must form a coherent summary when read alone.
3. Citations live on the same page as their reference — zero page-flipping.
4. Every visual must pass the Replacement Test and the Standalone Test.
5. The joke serves the truth; the truth does not serve the joke.
